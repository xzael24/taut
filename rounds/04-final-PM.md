### Remaining Concerns (Watch Items, Not Blockers)

These are items to monitor during execution, not reasons to delay:

1. **SMS costs at scale** — $200–500/month for 45K SMS at Phase 1 is manageable for MVP. The proposal has a mitigation (PIN-based auth for returning users to reduce OTP SMS), but this needs ongoing monitoring as the user base grows.

2. **3-month MVP timeline is aggressive** — Building an offline-first Android app with sync, SMS, encryption, audio feedback, kiosk mode AND deploying to 20 banks in 3 cities is tight. However, the scope is genuinely minimal, and the Phase 0 architecture spike buys down technical risk. The clear exit criteria provide a responsible off-ramp.

3. **Pengepul anchor commitment carries execution risk** — Getting a major pengepul to mandate TAUT is a high-leverage but high-difficulty negotiation. The mitigation (start with smaller/independent pengepul, offer exclusive supply data access) is sound, but this should be the PM's #1 priority during Phase 0.

4. **Auto-report value proposition still unvalidated** — The core hypothesis ("operator will switch because auto-report saves 3h/month") is reasonable but untested. That's exactly what the MVP is designed to validate. No change needed — this is working as intended.

5. **Grant-writing timeline** — CSR/grants take 6–12 months to secure. Phase 0 includes grant research, but a dedicated partnership/grant manager should be hired before Month 2. This is noted in the proposal but not concretely budgeted as a headcount. Recommend clarifying this in the technical breakdown.
